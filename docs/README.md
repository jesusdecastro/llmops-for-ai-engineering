# Documentación del proyecto

Este directorio centraliza la documentación funcional y técnica.

## Estructura

- `getting-started/`
  - `QUICKSTART.md`: arranque en pocos minutos
  - `SETUP.md`: instalación y configuración detallada
- `architecture/`
  - `ARCHITECTURE.md`: arquitectura y decisiones de diseño
- `deployment/`
  - `infraestructura_aws.md`: documentación de infraestructura/despliegue en AWS
- `project/`
  - `PROJECT_SUMMARY.md`: resumen técnico del proyecto
- Documentos de formación/onboarding ya existentes
  - `plan_formativo.md`
  - `copilot_sdd_onboarding.md` y recursos asociados

## Convención

- Nuevos documentos deben agregarse bajo una carpeta temática dentro de `docs/`.
- La raíz del repositorio debe mantener solo archivos imprescindibles para ejecución/configuración y un `README.md` principal.
